[ useSocket.js ]

Socket.io 연결, 이벤트 등록/해제, 룸 입장/퇴장 로직을
컴포넌트마다 반복해서 쓰면 코드가 길고 복잡해져 유지 보수성이랑 가독성이 떨어짐

이 파일은 그 복잡한 소켓 관련 로직을 "커스텀 훅"으로 만들어서
EditorPage에서 useSocket(roomId) 한 줄만 써도 모든 기능을 쓸 수 있게 해줌



[ 핵심 용어 설명 ]

▶ Hook (훅)
   - React 함수 컴포넌트에서 상태와 생명주기를 다루는 함수
   - use로 시작하는 함수 (예: useState, useEffect)
   - 커스텀 훅 = 직접 만드는 훅 = 반복되는 로직을 재사용 가능하게 포장

▶ useEffect
   - 컴포넌트가 화면에 나타났을 때(마운트) 또는 사라질 때(언마운트) 실행
   - 소켓 연결은 컴포넌트 마운트 시 시작, 언마운트 시 종료해야 함
   - return () => { 정리코드 } = 컴포넌트가 사라질 때 실행되는 클린업

▶ useRef
   - 렌더링에 영향을 주지 않는 값을 저장하는 상자
   - useState와 달리 값이 바뀌어도 리렌더링 안 됨
   - 소켓 객체는 state가 아닌 ref에 저장 (소켓 바뀔 때 렌더링 불필요)
   - socketRef.current = 실제 저장된 값에 접근

▶ useCallback
   - 함수를 메모이제이션(기억)해서 불필요한 재생성 방지
   - 의존성 배열이 바뀌지 않으면 같은 함수 참조를 유지
   - 자식 컴포넌트에 함수를 props로 전달할 때 성능 최적화에 유용

▶ 마운트 / 언마운트
   - 마운트: 컴포넌트가 화면에 처음 나타남
   - 언마운트: 컴포넌트가 화면에서 사라짐 (다른 페이지로 이동 등)

▶ 의존성 배열 [roomId]
   - useEffect의 두 번째 인자
   - 배열 안의 값이 바뀌면 useEffect 재실행
   - [] (빈 배열) = 마운트 시 한 번만 실행
   - [roomId] = roomId가 바뀔 때마다 재실행

▶ Optional Chaining (?.)
   - socketRef.current?.emit(...)
   - socket이 null이나 undefined여도 에러 없이 그냥 무시
   - 없으면 undefined 반환
